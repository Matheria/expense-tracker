# API Reference

Base URL: `http://localhost:3001/api`  
Swagger UI: `http://localhost:3001/docs`

Все защищённые эндпоинты требуют заголовка:
```
Authorization: Bearer <accessToken>
```

---

## Auth

### POST /auth/register

Регистрация нового пользователя.

**Тело запроса:**
```json
{
  "email": "user@example.com",
  "password": "минимум8символов",
  "name": "Иван"
}
```

| Поле | Тип | Обязательно | Ограничения |
|---|---|---|---|
| `email` | string | да | валидный email |
| `password` | string | да | 8–72 символа |
| `name` | string | нет | макс. 100 символов |

**Ответ `201`:**
```json
{ "accessToken": "<jwt>" }
```

**Ошибки:**
- `400` — ошибка валидации
- `409` — email уже занят

---

### POST /auth/login

Вход в аккаунт.

**Тело запроса:**
```json
{
  "email": "user@example.com",
  "password": "пароль"
}
```

**Ответ `200`:**
```json
{ "accessToken": "<jwt>" }
```

**Ошибки:**
- `400` — ошибка валидации
- `401` — неверные учётные данные или неактивный аккаунт

## Users

### GET /users/me `🔒`

Профиль текущего пользователя.

**Ответ `200`:**
```json
{
  "id": "uuid",
  "name": "Иван",
  "email": "user@example.com"
}
```

**Ошибки:**
- `401` — токен отсутствует или недействителен
- `404` — пользователь не найден (токен валиден, но запись удалена)

## Categories

### POST /categories `🔒`

Создать категорию.

**Тело запроса:**
```json
{
  "name": "Продукты",
  "color": "#FF5733",
  "icon": "shopping-cart"
}
```

| Поле | Тип | Ограничения |
|---|---|---|
| `name` | string | не пустая, макс. 50 символов |
| `color` | string | формат `#RRGGBB` |
| `icon` | string | не пустая, макс. 50 символов |

**Ответ `201`:** объект категории.

**Ошибки:** `400`, `401`

---

### GET /categories `🔒`

Список всех категорий текущего пользователя.

**Ответ `200`:**
```json
[
  {
    "id": "uuid",
    "name": "Продукты",
    "color": "#FF5733",
    "icon": "shopping-cart",
    "userId": "uuid",
    "createdAt": "2024-01-15T10:30:00.000Z",
    "updatedAt": "2024-01-15T10:30:00.000Z"
  }
]
```

**Ошибки:** `401`

---

### PATCH /categories/:id `🔒`

Частичное обновление категории. Все поля опциональны.

**Тело запроса:** любое подмножество полей `CreateCategoryDto`.

**Ответ `200`:** обновлённый объект категории.

**Ошибки:** `400`, `401`, `404` (категория не найдена или чужая)

---

### DELETE /categories/:id `🔒`

Удалить категорию.

**Ответ `204`:** пустое тело.

**Ошибки:**
- `401` — нет токена
- `404` — категория не найдена или чужая
- `409` — к категории привязаны транзакции (нужно сначала удалить их)

## Transactions

### POST /transactions `🔒`

Создать транзакцию.

**Тело запроса:**
```json
{
  "amount": 1500.50,
  "type": "EXPENSE",
  "description": "Обед в кафе",
  "date": "2024-01-15",
  "categoryId": "uuid"
}
```

| Поле | Тип | Обязательно | Ограничения |
|---|---|---|---|
| `amount` | number | да | > 0, макс. 2 знака после запятой |
| `type` | `INCOME` \| `EXPENSE` | да | enum |
| `description` | string | нет | не пустая, макс. 255 символов |
| `date` | string | да | ISO 8601 |
| `categoryId` | string | да | UUID; категория должна принадлежать пользователю |

**Ответ `201`:** объект транзакции (`amount` — строка, т.к. `Decimal`).

**Ошибки:** `400`, `401`, `404` (категория не найдена или чужая)

---

### GET /transactions `🔒`

Список транзакций с фильтрацией и пагинацией.

**Query-параметры:**

| Параметр | Тип | Default | Описание |
|---|---|---|---|
| `dateFrom` | ISO 8601 | — | Начало диапазона (включительно) |
| `dateTo` | ISO 8601 | — | Конец диапазона (включительно) |
| `type` | `INCOME` \| `EXPENSE` | — | Фильтр по типу |
| `categoryId` | UUID | — | Фильтр по категории |
| `page` | number ≥ 1 | `1` | Номер страницы |
| `limit` | number 1–100 | `10` | Записей на странице |

**Ответ `200`:**
```json
{
  "data": [ /* Transaction[] */ ],
  "total": 42,
  "page": 1,
  "limit": 10,
  "totalPages": 5
}
```

**Ошибки:** `400`, `401`

---

### GET /transactions/summary `🔒`

Финансовая сводка за месяц.

**Query-параметры:**

| Параметр | Тип | Описание |
|---|---|---|
| `month` | number 1–12 | Номер месяца |
| `year` | number ≥ 2000 | Год |

**Ответ `200`:**
```json
{
  "month": 6,
  "year": 2024,
  "income": 50000,
  "expense": 35000,
  "balance": 15000,
  "byCategory": [
    { "categoryId": "uuid", "type": "EXPENSE", "total": 3500 }
  ]
}
```

**Ошибки:** `400`, `401`

---

### GET /transactions/:id `🔒`

Получить транзакцию по ID.

**Ответ `200`:** объект транзакции.

**Ошибки:** `401`, `404`

---

### PATCH /transactions/:id `🔒`

Частичное обновление транзакции. Все поля опциональны (те же, что у `POST`).

**Ответ `200`:** обновлённый объект транзакции.

**Ошибки:** `400`, `401`, `404`

---

### DELETE /transactions/:id `🔒`

Удалить транзакцию.

**Ответ `204`:** пустое тело.

**Ошибки:** `401`, `404`

## Формат объекта Transaction

```json
{
  "id": "uuid",
  "amount": "1500.50",
  "type": "EXPENSE",
  "description": "Обед в кафе",
  "date": "2024-01-15T00:00:00.000Z",
  "categoryId": "uuid",
  "userId": "uuid",
  "createdAt": "2024-01-15T10:30:00.000Z"
}
```

> `amount` — строка (`Prisma.Decimal` сериализуется в строку). На фронтенде тип `string`.

---

## Коды ошибок

| Код | Причина |
|---|---|
| `400` | Ошибка валидации (`class-validator`) |
| `401` | Токен отсутствует, истёк или недействителен |
| `404` | Ресурс не найден или принадлежит другому пользователю |
| `409` | Конфликт: email занят / есть связанные записи |
